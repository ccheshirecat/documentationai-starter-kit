## Branding and Themes

Tailor HYPR's appearance to reflect your brand. You edit theme settings via the configuration file or admin UI.

<Steps>
  <Step title="Access Theme Editor" icon="settings">
    Navigate to Admin > Themes in the dashboard.
  </Step>
  <Step title="Update Colors" icon="palette">
    Set primary color to `#3B82F6` and adjust secondary shades.
  </Step>
  <Step title="Upload Logo" icon="image">
    Replace default logo with your SVG or PNG file (max 200x50px).
  </Step>
  <Step title="Preview and Save" icon="eye">
    Preview changes live, then save to deploy.
  </Step>
</Steps>

Use a `theme.json` file for advanced customization:

<CodeGroup tabs="JSON,YAML">
  ```json
  {
    "primaryColor": "#3B82F6",
    "logoUrl": "https://your-domain.com/logo.svg",
    "fontFamily": "Inter, sans-serif",
    "sidebarWidth": "280px"
  }
  ```
  ```yaml
  primaryColor: "#3B82F6"
  logoUrl: https://your-domain.com/logo.svg
  fontFamily: Inter, sans-serif
  sidebarWidth: 280px
  ```
</CodeGroup>

## User Access Controls

Manage who views and edits your documentation. Define roles like Viewer, Editor, and Admin with granular permissions.

<Tabs>
  <Tab title="UI Setup" icon="monitor">
    In Admin > Users, assign roles:

    | Role   | View | Edit | Delete | Admin |
    |--------|------|------|--------|-------|
    | Viewer | ✅   | ❌   | ❌     | ❌    |
    | Editor | ✅   | ✅   | ❌     | ❌    |
    | Admin  | ✅   | ✅   | ✅     | ✅    |

    Select users and apply roles instantly.
  </Tab>
  <Tab title="API Configuration" icon="api">
    Use the permissions endpoint to script access:

    <Request tabs="cURL,JavaScript" show-lines="true">
      ```bash
      curl -X POST https://api.hyprdocs.com/v1/permissions \
        -H "Authorization: Bearer YOUR_API_KEY" \
        -H "Content-Type: application/json" \
        -d '{
          "userId": "user-123",
          "role": "editor",
          "pages": ["docs/*"]
        }'
      ```
      ```javascript
      const response = await fetch('https://api.hyprdocs.com/v1/permissions', {
        method: 'POST',
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY',
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          userId: 'user-123',
          role: 'editor',
          pages: ['docs/*']
        })
      });
      ```
    </Request>
  </Tab>
</Tabs>

<Callout kind="alert">
  Restrict admin roles to trusted users to prevent accidental deletions.
</Callout>

## Export and Import Options

Transfer documentation between projects or backups effortlessly.

<ExpandableGroup>
  <Expandable title="Export to Markdown/ZIP" default-open="true">
    Export selects pages as Markdown files or ZIP archives.

    ```bash
    hypr export --project my-docs --format md --output ./backup/
    ```

    This generates folder-structured Markdown ready for Git or other CMS.
  </Expandable>
  <Expandable title="Import from GitHub">
    Pull content from a repo:

    ```bash
    hypr import --repo owner/my-docs-repo --branch main --path docs/
    ```

    Maps Markdown files to HYPR pages automatically.
  </Expandable>
</ExpandableGroup>

<ParamField path="format" param-type="string" required="true">
  Export format: `md`, `html`, or `zip`.
</ParamField>

<ParamField query="includeAttachments" param-type="boolean" required="false">
  Set to `true` to bundle images and assets.
</ParamField>

These tools ensure your HYPR setup remains flexible and portable across teams and platforms.
